To favor the readability and the intrinsic understandability of your models, pluralize the name of array properties.

Noncompliant Code Example (OpenAPI 3)

openapi: "3.0.1"
info:
  version: 1.0.0
  title: Swagger Petstore
components:
  schemas:
    Car:
      properties:
        feature:              # Noncompliant: Use plural for array properties.
          type: array
          items:
            type: string

Compliant Solution (OpenAPI 3)

openapi: "3.0.1"
info:
  version: 1.0.0
  title: Swagger Petstore
components:
  schemas:
    Car:
      properties:
        features:
          type: array
          items:
            type: string
