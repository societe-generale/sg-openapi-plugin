Use only camel-case for query parameters and fields of input/output JSON structures. This increases the readability of the payloads in your API's input and output values. Consistent usage of the same style also improves the general accessibility of your API by making its usage more intuitive.

Noncompliant Code Example

{
    "some-field": "some value",
    "other_field": 1
}

Compliant Solution

{
    "someField": "some value",
    "otherField": 1
}
